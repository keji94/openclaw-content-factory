## 发布到微信公众号

**Markdown 转 HTML** (默认 AI 模式，已通过环境变量配置):
```bash
md2wechat convert article.md --json
```

**🎨 AI 生成封面图**:
```bash
cd ~/.openclaw/workspace-content/skills/ai-cover-generator
python generate_cover.py --title "文章标题" --output ./cover.jpg --json
```
**上传图片**:
```bash
md2wechat upload_image ./cover.jpg --json
```

**创建草稿** (⚠️ JSON 必须包裹在 articles 数组中):
```bash
md2wechat create_draft draft.json --json
```
draft.json 格式：
```json
{
  "articles": [{
    "title": "文章标题",
    "content": "<p>HTML内容</p>",
    "thumb_media_id": "封面图的media_id（从upload_image获取）",
    "digest": "文章摘要（可选）",
    "author": "作者（可选）"
  }]
}
```

**查看主题**:
```bash
md2wechat themes --json
```

## ⚠️ 字段写入注意事项

### URL 类型字段（原文链接、链接、知识库链接）
飞书 API 要求对象格式，不能直接传字符串：
```json
"原文链接": {"link": "https://xxx.com", "text": "查看原文"}
```
❌ 错误：`"原文链接": "https://xxx.com"` → 会报 `URLFieldConvFail`

### 人员字段
格式：`[{id: "ou_xxx"}]`（数组对象）

### 日期字段
格式：毫秒时间戳（如 `1741977600000`），不是秒，也不是字符串

### 单选字段
格式：字符串（如 `"待处理"`），不是数组

### 多选字段
格式：字符串数组（如 `["#方法", "#案例"]`）

## 使用流程

1. **存素材&生成选题**：用户发链接 → AI 读取内容 → 创建素材库记录
2. **推选题**：AI 查询素材库 → 筛选待处理 → 推荐给用户
3. **建档案**：用户选选题 → AI 创建内容写作记录 → 出大纲
4. **写初稿**：用户选大纲 → AI 创建云文档 → 生成初稿
5. **定稿归档**：用户确认定稿 → AI 移动到知识库 → 更新状态
