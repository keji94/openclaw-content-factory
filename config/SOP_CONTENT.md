# SOP_CONTENT.md · 内容生产标准流程

## 流程概览

```
灵感入库 → 选题推荐 → 大纲生成 → 初稿写作 → 修改定稿 → 归档发布
```

---

## 场景1：灵感入库

### 触发条件
用户发送的消息中包括链接或内容片段。

### 执行步骤

1. 读取内容
   - 如果是链接，使用playwright-scraper隐私模式能力或者tavily skill读取网页内容
   - 如果是抖音/小红书/B站，提取视频/帖子信息
   - 读取成功后通知用户
2. 提取信息
   - 标题
   - 来源
   - 原文链接
   - 内容摘要（200-400字）
   - 标签（2-3合适的标签）

3. 创建灵感记录
   强制要求：一个潜在选题一条记录，按优先级从高到低排列
   在「灵感库」子表创建记录：
   - 标题: [提取的标题]
   - 来源: [识别的来源]
   - 原文链接: [用户发的链接] (必须填充)
   - 内容摘要: [AI提炼的摘要]（200-400字）
   - 标签: [AI自动打标签]
   - 状态: 待处理
   - 优先级:高、中、低
   - 潜在选题: [AI拆解的选题方向] (一个选题一行记录)
   - 选题理由：[AI分析做这个选题的理由]
   - 创建时间

4. 反馈用户
   ```
   ✅ 已存入灵感库
   
   📌 标题：xxx
   🏷️ 标签：#选题 #素材
   💡 潜在选题：
   - 选题1（优先级）
   - 选题2（优先级）
   ```

---

## 场景2：选题推荐

### 触发条件
用户发送「推荐选题」、「有什么选题」、「写点什么」

### 执行步骤

1. 查询灵感库
   ```
   筛选条件：状态 = 待处理
   排序：优先级 高 > 中 > 低
   ```

2. 生成推荐
   ```
   📋 推荐选题（共 X 个待处理）
   
   | 序号 | 标题 | 来源 | 优先级 | 推荐理由 |
   |------|------|------|--------|----------|
   | 1 | xxx | 抖音 | 高 | ... |
   | 2 | xxx | 小红书 | 中 | ... |
   ```

3. 等待用户选择

---

## 场景3：大纲生成

### 触发条件
用户选择某个选题，如「选1」或「写第一个」或「选题关键字」

### 执行步骤

1. 创建内容记录
   ```
   在「内容写作」子表创建记录：
   - 标题: [选题标题]
   - 选题来源: [关联灵感库记录]
   - 阶段: 选题
   ```

2. 生成大纲选项
   ```
   📝 选题：xxx
   
   我为你准备了 3 个大纲方向：
   
   【方向A】xxx
   1. ...
   2. ...
   3. ...
   
   【方向B】xxx
   1. ...
   2. ...
   
   【方向C】xxx
   1. ...
   
   请选择一个方向，或告诉我你想调整什么。
   ```
3. 确定文章结构
   3.1 基于用户选定的大纲和大纲方向，生成三种文章结构。
   3.2 询问用户是否需要修改文章结构

4. 更新记录
   ```
   更新「内容写作」记录：
   - 阶段: 大纲
   - 大纲: [用户选择的大纲]
   ```

---

## 场景4：初稿写作

### 触发条件
用户确定大纲文章结构

### 执行步骤

1. 创建云文档
   ```
   在飞书云文档创建新文档：
   - 标题: [文章标题]
   ```

2. 生成初稿
   - 根据大纲生成完整文章，完成后通知用户状态
   - 进行准确性校验，开始前完成后都通知用户状态
   - 进行人性化改写，开始前完成后都通知用户状态
   - 写入云文档

3. 回填链接
   ```
   更新「内容写作」记录：
   - 阶段: 初稿
   - 链接: [云文档链接]
   ```

4. 反馈用户
   ```
   ✅ 初稿已完成！
   
   📄 文档链接：[云文档链接]
   
   请查看并修改，完成后告诉我「定稿」。
   ```

---

## 场景5：修改定稿

### 触发条件
用户说「定稿」或「完成了」

### 执行步骤

1. 确认定稿
   ```
   更新「内容写作」记录：
   - 阶段: 定稿
   ```

2. 更新灵感库
   ```
   更新关联的灵感库记录：
   - 状态: 已写作
   ```

---

## 场景6：归档发布

### 触发条件
用户确认发布或要求归档

### 执行步骤

1. 移动到知识库
   ```
   将云文档移动到知识库：
   - 目标目录: /已发布文章/[自动分类]/
   ```

2. 更新记录
   ```
   更新「内容写作」记录：
   - 阶段: 已发布
   - 知识库链接: [知识库链接]
   - 发布时间: [当前时间]
   
   更新「灵感库」记录：
   - 状态: 已发布
   ```

3. 反馈用户
   ```
   🎉 已归档到知识库！
   
   📁 目录：[分类目录]
   🔗 链接：[知识库链接]
   
   本次内容生产完成 ✅
   ```

---

## 异常处理

### 用户中途取消
```
更新「内容写作」记录：
- 阶段: 已取消
- 备注: 用户取消原因
```

### 需要补充信息
```
主动询问用户，不要猜测
记录用户的回答到 MEMORY.md
```

### 权限不足
```
明确告诉用户需要开通什么权限
给出操作指引
```
