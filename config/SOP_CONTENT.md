# SOP_CONTENT.md · 内容生产标准流程

## 流程概览

```
素材入库&选题生成 → 选题推荐 → 大纲生成 → 初稿写作 → 智能审稿 → 润色打磨 → 定稿归档 → 归档发布
```

---

## ⚠️ 执行强制检查点

完成每个场景后，必须自问：

1. **当前场景是否完成？**
2. **SOP 里定义的下一个场景是什么？**
3. **是否需要自动触发下一个场景？**

**禁止在 SOP 定义的流程未完成前，向用户反馈「完成」。**

**特别说明**：
- 场景4（初稿写作）完成后，必须自动触发场景5（智能审稿）
- 场景5（智能审稿）完成后，必须自动触发场景6（润色打磨，调用 humanizer-zh）
- 只有场景6完成后，才能向用户反馈「初稿完成，请审阅」

## 场景1：素材入库&选题生成

### 触发条件
用户发送的消息中包括链接或内容片段。

### 执行步骤

**Step 1: 内容抓取**
- 抖音/小红书/B站 → 使用 playwright-scraper 或对应 skill
- 公众号文章 → 使用 playwright-scraper 隐私模式
- 普通网页 → 使用 web_fetch
- 抓取成功后通知用户

**Step 2: 信息提取**
- 标题（原文标题）
- 来源（识别平台）
- 原文链接
- 内容摘要（200-400字核心要点）
- 标签（2-3个合适标签）
- 逐字稿：**如果是视频解析，原视频的逐字稿必须完整保存，一字不省** ⚠️ 逐字稿是花费 token 提取的有价值内容，是后续写作的原始素材

**Step 3: 选题拆解（核心）**
**强制要求**：从一篇素材拆解出 1-3 个不同角度的选题，每个选题一条记录。

选题拆解原则：
- 不同受众视角（新手 vs 进阶 vs 专家）
- 不同内容形式（教程 vs 观点 vs 案例）
- 不同切入点（痛点型 vs 方法型 vs 趋势型）

**选题价值模板**：
```
【选题标题】：xxx
【目标受众】：xxx
【核心价值】：xxx
【差异化】：与其他同类内容的不同点
【时效性】：是否需要紧跟热点
```


**Step 4: 写入飞书**

先获取字段类型（`feishu_bitable_app_table_field.list`），按格式写入：

```json
{
  "标题": "原文标题",
  "来源": "抖音/小红书/公众号/网页",
  "原文链接": {"link": "URL", "text": "查看原文"},
  "内容摘要": "AI提炼的摘要",
  "标签": ["#选题", "#素材"],
  "状态": "待处理",
  "优先级": "高/中/低",
  "选题标题": "从原文拆解的具体选题（必填）",
  "选题价值": "做这个选题的理由、价值、受众分析（必填）",
   "逐字稿": "如果是视频，必须保存逐字稿"
}
```

**Step 5: 反馈用户**
```
✅ 已从原文拆解出 X 个选题

📌 原文标题：xxx
🏷️ 标签：#选题 #素材

💡 拆解出的选题：
1. 【选题标题1】（优先级：高）
   受众：xxx | 价值：xxx
2. 【选题标题2】（优先级：中）
   受众：xxx | 价值：xxx

共入库 X 条选题记录。
```

## 场景3：大纲生成

### 触发条件
用户选择某个选题（「选1」「写第一个」「选题关键字」）

### 执行步骤

**Step 1: 创建内容记录**
```json
{
  "标题": "选题标题",
  "选题来源": ["关联的素材库记录ID"],
  "阶段": "选题"
}
```

**Step 2: 证据池构建**

如果选题需要外部素材支撑：
1. 使用 WebSearch 搜索相关内容
2. 抓取 3-5 篇参考文章
3. 整理成证据池

**证据池格式**：
```
## 证据池

### 来源1：xxx
- 标题：xxx
- 链接：xxx
- 发布时间：xxx
- 来源类型：official/community/practice
- 核心要点：
  1. xxx
  2. xxx
- 可信度：high/medium/low
```

**Step 3: 生成大纲选项**

**⚠️ 避免洗稿的硬性要求**：

1. **禁止照搬原文结构**——不能逐条罗列原文要点
2. **必须重构框架**——归纳、合并、换个角度切入
3. **必须加入原创观点**——结合自己的理解、经验或补充案例
4. **引用要标注来源**——文末必须附参考资料链接

提供 3 个不同方向的大纲：

```
📝 选题：xxx

我为你准备了 3 个大纲方向：

【方向A】痛点解决型（适合新手）
1. 开篇：场景引入 + 痛点共鸣
2. 分析：为什么会这样
3. 方法：3-5个可执行步骤
4. 案例：实际应用场景
5. 总结：行动清单

【方向B】观点输出型（适合进阶）
1. 开篇：抛出观点
2. 论证1：数据/案例支撑
3. 论证2：对比分析
4. 论证3：实践经验
5. 总结：核心观点 + 延伸思考

【方向C】案例复盘型（适合专业）
1. 背景：项目/事件介绍
2. 过程：关键节点复盘
3. 问题：遇到的坑
4. 方案：如何解决
5. 方法论：可复用的经验

请选择一个方向，或告诉我你想调整什么。
```

**Step 4: 更新记录**
```json
{
  "阶段": "大纲",
  "大纲": "用户选择的大纲内容"
}
```

## 场景4：初稿写作（核心）

### 触发条件
用户确定大纲方向

### 执行步骤

**Step 1: 创建云文档**
```
在飞书云文档创建新文档，标题为文章标题
```

**Step 2: 生成初稿**

**写作规范**：

**标题技巧**：
- 数字型：`5个方法让你...`
- 提问型：`为什么...？`
- 对比型：`A与B的区别...`
- 悬念型：`你不知道的...`

**开头要求**（前3-5行）：
- 提出痛点或引发共鸣
- 设置悬念或提出问题
- 明确文章价值

**正文结构**：
- 使用小标题分段（## 或 ###）
- 每段 200-300 字
- 包含案例、数据支撑
- 使用列表增强可读性

**配图流程**：如需配图，先生成图片获取 token，创建文档时就包含 `<image token="xxx"/>` 标记（图片无法后续追加到指定位置）

**结尾要求**：
- 总结核心观点
- 引导互动（点赞、在看、评论）
- 可添加金句或行动呼吁

**字数要求**：1500-2500 字


**Step 3: 可追溯性检查**

确保每个关键论点都能在证据池或素材中找到支撑。文章末尾添加：

```
## 参考资料

- [来源1标题](链接) - 官方文档
- [来源2标题](链接) - 社区讨论
- [来源3标题](链接) - 实践案例
```


**Step 4: 写入云文档并回填链接**
```json
{
  "阶段": "初稿",
  "链接": {"link": "云文档URL", "text": "查看文档"}
}
```

**Step 5: 反馈用户**
```
✅ 初稿已完成！

📄 文档链接：[云文档链接]
📊 字数统计：xxx 字

请查看并修改，需要我帮你审稿吗？。
```

## 场景5：智能审稿（来自 yuan-skills）

### 触发条件
初稿完成后，自动触发或在用户要求时触发

### 审稿四维度

| 维度 | 检查要点 | 权重 |
|------|---------|------|
| 逻辑 | 论点清晰、论据充分、推理合理 | 30% |
| 表达 | 语言流畅、无AI痕迹、口语化适度 | 25% |
| 数据 | 数据准确、案例恰当、引用规范 | 25% |
| 结构 | 标题吸引、段落分明、首尾呼应 | 20% |

### 硬检查项

1. **可追溯性**：关键论断是否在证据池中有支撑
2. **标题一致性**：标题承诺是否在正文明确兑现

### 评分标准

- 90-100 分：优秀，可直接发布
- 80-89 分：良好，小幅优化即可
- 70-79 分：合格，需要修改
- 70 分以下：需要重写

### 审稿报告格式

```
📋 审稿报告

综合评分：85/100

【逻辑 26/30】
✅ 论点清晰，结构完整
⚠️ 第三段论证略显单薄，建议补充案例

【表达 22/25】
✅ 整体流畅
⚠️ 发现 AI 痕迹词汇：「综上所述」「不难看出」

【数据 20/25】
✅ 数据来源可信
⚠️ 第二个数据缺少出处

【结构 17/20】
✅ 段落分明
⚠️ 结尾略显仓促

📝 修改建议：
1. 替换 AI 痕迹词汇
2. 补充第三段案例
3. 添加数据来源
4. 优化结尾引导
```

### 关键命令速查

```bash
# 1. Markdown 转 HTML（获取 prompt）
md2wechat convert article.md --mode ai --theme autumn-warm --json

# 2. 🎨 AI 生成封面图
cd ~/.openclaw/workspace-content/skills/ai-cover-generator
python generate_cover.py --title "文章标题" --output ./cover.jpg --json

# 3. 上传图片
md2wechat upload_image ./cover.jpg --json
md2wechat download_and_upload https://example.com/image.jpg --json

# 4. 创建草稿
md2wechat create_draft draft.json --json

# 5. 查看可用主题
md2wechat themes --json
```

### 封面图生成说明

**自动风格匹配**：根据文章标题关键词自动选择风格

| 关键词 | 风格 |
|--------|------|
| AI、人工智能 | 科技感，蓝色调，神经网络图案 |
| 教程、如何 | 简约现代风格，步骤图示 |
| 案例、复盘 | 商务专业风格，数据图表 |
| 观点、思考 | 抽象艺术风格，对比色块 |
| 方法、技巧 | 实用工具风格，技巧提示 |

**封面图尺寸**：默认 900x383 (2.35:1)，公众号官方推荐封面大图尺寸

### 注意事项

1. **封面图片**：创建草稿必须有封面图片
   - 优先使用用户提供的封面
   - 若无封面，AI 自动生成
   - 需要配置 SILICONFLOW_API_KEY
2. **图片上传**：微信图片需先上传到素材库获取 media_id
3. **IP白名单**：服务器 IP 需要在微信公众号后台添加到白名单
4. **审核时间**：草稿创建后可在公众号后台预览和发布

### 异常处理

| 异常情况 | 处理方式 |
|---------|---------|
| 未配置微信凭证 | 提示用户配置 WECHAT_APPID/SECRET |
| 未配置硅基流动 API Key | 提示用户配置 SILICONFLOW_API_KEY 或手动提供封面 |
| IP 不在白名单 | 告知需要添加的 IP 地址 |
| 封面图生成失败 | 重试 3 次，失败则提示用户手动上传 |
| 图片上传失败 | 跳过该图片，使用占位符，后续手动替换 |

## 异常处理

| 异常情况 | 处理方式 |
|---------|---------|
| 用户中途取消 | 更新阶段为「已取消」，记录原因 |
| 需要补充信息 | 主动询问，记录到 MEMORY.md |
| 权限不足 | 明确告知需要开通的权限 |

| 参考文章无法访问 | 跳过该 URL，继续处理其他素材 |
| 初稿质量过低 | 重新生成或提示用户提供更多素材 |
| 审稿评分 < 70 | 建议用户检查主题是否合适 |
