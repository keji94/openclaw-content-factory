# 场景3：大纲生成

## 触发条件
用户选择某个选题（「选1」「写第一个」「选题关键字」）

## 执行步骤

**Step 1: 创建内容记录**
```json
{
  "标题": "选题标题",
  "选题来源": ["关联的素材库记录ID"],
  "阶段": "选题"
}
```

**Step 2: 证据池构建**

如果选题需要外部素材支撑：
1. 使用 WebSearch 搜索相关内容
2. 抓取 3-5 篇参考文章
3. 整理成证据池

**证据池格式**：
```
## 证据池

### 来源1：xxx
- 标题：xxx
- 链接：xxx
- 发布时间：xxx
- 来源类型：official/community/practice
- 核心要点：
  1. xxx
  2. xxx
- 可信度：high/medium/low
```

**Step 3: 生成大纲选项**

**⚠️ 避免洗稿的硬性要求**：

1. **禁止照搬原文结构**——不能逐条罗列原文要点
2. **必须重构框架**——归纳、合并、换个角度切入
3. **必须加入原创观点**——结合自己的理解、经验或补充案例
4. **引用要标注来源**——文末必须附参考资料链接

提供 3 个不同方向的大纲：

```
📝 选题：xxx

我为你准备了 3 个大纲方向：

【方向A】痛点解决型（适合新手）
1. 开篇：场景引入 + 痛点共鸣
2. 分析：为什么会这样
3. 方法：3-5个可执行步骤
4. 案例：实际应用场景
5. 总结：行动清单

【方向B】观点输出型（适合进阶）
1. 开篇：抛出观点
2. 论证1：数据/案例支撑
3. 论证2：对比分析
4. 论证3：实践经验
5. 总结：核心观点 + 延伸思考

【方向C】案例复盘型（适合专业）
1. 背景：项目/事件介绍
2. 过程：关键节点复盘
3. 问题：遇到的坑
4. 方案：如何解决
5. 方法论：可复用的经验

请选择一个方向，或告诉我你想调整什么。
```

**Step 4: 更新记录**
```json
{
  "阶段": "大纲",
  "大纲": "用户选择的大纲内容"
}
```
