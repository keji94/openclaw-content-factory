# 场景4：初稿写作（核心）

## 触发条件
用户确定大纲方向

## 执行步骤

**Step 1: 创建云文档**
```
在飞书云文档创建新文档，标题为文章标题
```

**Step 2: 生成初稿**

**写作规范**：

**标题技巧**：
- 数字型：`5个方法让你...`
- 提问型：`为什么...？`
- 对比型：`A与B的区别...`
- 悬念型：`你不知道的...`

**开头要求**（前3-5行）：
- 提出痛点或引发共鸣
- 设置悬念或提出问题
- 明确文章价值

**正文结构**：
- 使用小标题分段（## 或 ###）
- 每段 200-300 字
- 包含案例、数据支撑
- 使用列表增强可读性

**配图流程**：如需配图，先生成图片获取 token，创建文档时就包含 `<image token="xxx"/>` 标记（图片无法后续追加到指定位置）

**结尾要求**：
- 总结核心观点
- 引导互动（点赞、在看、评论）
- 可添加金句或行动呼吁

**字数要求**：1500-2500 字


**Step 3: 可追溯性检查**

确保每个关键论点都能在证据池或素材中找到支撑。文章末尾添加：

```
## 参考资料

- [来源1标题](链接) - 官方文档
- [来源2标题](链接) - 社区讨论
- [来源3标题](链接) - 实践案例
```


**Step 4: 写入云文档并回填链接**
```json
{
  "阶段": "初稿",
  "链接": {"link": "云文档URL", "text": "查看文档"}
}
```

**Step 5: 保存初稿快照（写作学习）**

保存初稿用于后续对比学习：
```bash
python scripts/observe_draft.py \
    --article-id "YYYY-MM-DD_标题关键词" \
    --title "文章标题" \
    --source "云文档URL" \
    --content "初稿MD内容"
```
> article-id 格式：日期_标题关键词（如 `2026-03-28_AI工具推荐`），Scene 7 定稿时需用同一 ID

**Step 6: 反馈用户**
```
✅ 初稿已完成！

📄 文档链接：[云文档链接]
📊 字数统计：xxx 字

请查看并修改，需要我帮你审稿吗？。
```
