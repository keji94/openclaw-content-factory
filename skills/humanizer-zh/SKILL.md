# Humanizer-ZH - 中文内容人性化改写器

专业的中文内容人性化改写工具，专注于降低AIGC检测分数，让AI生成内容更像人类写作。

## 📊 当前版本

**Version**: 1.0.0

**核心能力**：
- 降低AIGC检测分数（目标：AIGC < 0.20）
- 59个专业改写策略（Z1-Z59）
- 多平台检测经验整合（朱雀、ContentAny、ai-detector v5.6）
- 多种文体支持（小说剧情解说、技巧总结、公众号文章等）

---

## 🎯 改写模式

### Mode 1: fix_aigc - 标准AIGC修复
**适用**: AIGC 0.50-0.70
**策略**: Level 2改写（结构调整）
**预期**: AIGC降低15-25%

### Mode 2: aggressive - 激进改写
**适用**: AIGC > 0.70 或朱雀检测 > 0.80
**策略**: Level 3改写（结构破坏）
**预期**: AIGC降低30-50%

### Mode 3: fix_analysis_content - 分析内容修复
**适用**: 分析类文章（大段落3类型）
**策略**: 从分析模式到叙述模式
**预期**: AIGC降低30-40%

## 📚 改写策略索引（Z1-Z59）

### 基础策略（Z1-Z10）

```yaml
策略Z1: 机械枚举结构破坏（朱雀极强特征，权重+0.8）
  识别: 5项以上列举 + 完全相同句式结构 + 节奏性平行
  改写: 打破为段落式分散叙述，删除功能式动词，改变节奏

策略Z2: 删除高AIGC段落（优于改写）
  识别: AIGC > 0.85 + 改写价值低 + 删除后不影响连贯性
  改写: 完全删除，在其他段落自然提及

策略Z3: 程序化表达彻底删除
  识别: "大概是...最..."、"太...了"、"抄得飞起"、"行走的XX库"
  改写: 完全删除程序化评价

策略Z4: 功能式动词在机械枚举中必须彻底删除
  识别: 机械枚举结构 + 功能式动词密集
  改写: 删除"完成"、"激发"、"促成"、"升华"、"承载"、"体现"

策略Z5: 保留自然口语化和个人声音
  识别: "你想想"、"结果呢?"、"傻眼了"、"奇怪的是"
  改写: 保留（朱雀不判为AI特征）

策略Z6: 4项对称枚举+破折号（ContentAny极强特征）
  识别: 4个对称项 + 双破折号包围 + 功能式动词结尾
  改写: 打破4项对称，删除双破折号，删除功能式动词

策略Z7: "每一X都有它的Y"模式（ContentAny极强特征）
  识别: 所有化表达 + 程序化概括 + 功能式动词
  改写: 删除套话模式，改为自然表达

策略Z8: "最关键的是"引导语（ContentAny强特征）
  识别: "最关键的是"、"重要的是"、"核心的是"
  改写: 删除程序化引导语，直接陈述内容

策略Z9: "这种X的处理"句式（ContentAny强特征）
  识别: "这种X的处理/让/使" + 程序化抽象概括
  改写: 删除"这种X的处理"，改为直接描述

策略Z10: 3项括号注释（ContentAny中等特征）
  识别: 3个以上括号注释 + 对称结构
  改写: 删除括号，改为独立句子，打破对称节奏
```

## 🔧 核心原则

### 原则1: 自然性优先

```yaml
自然性优先原则:
  优先级: 最高（高于所有其他规则）
  定义: "降低AIGC不能牺牲句子的自然度和可读性"

  检查标准:
    - 读起来是否流畅自然？
    - 是否像人类正常表达？
    - 会不会有生硬感？
    - 会不会让读者困惑？

  危险信号（过度删减）:
    - ❌ 两个名词直接连用："那汉子张荣"
    - ❌ 动词宾语不完整："统领梁山泊水匪"
    - ❌ 缺少谓语："他眉头紧锁"
    - ❌ 破句感：读起来感觉少了什么

  正确策略:
    - ✅ 通过"调整语序"而非"删减词汇"来优化
    - ✅ 保留必要的语法结构（"是"、"的"）
    - ✅ 合并冗余结构
    - ✅ 名字后置作为同位语
```

### 原则2: 功能式动词智能处理

```yaml
功能式动词智能处理:
  优先级: 与原则1同等（最高优先级）
  定义: "功能式动词不一定是AI特征，需智能判断"

  功能式动词列表:
    High-level: 成为、符合、体现、承载、完成、实现 (+0.4 each)
    Mid-level: 培养、放弃、处理、管理、带领、发展... (+0.3 each)
    Basic-level: 擅长、担任、负责、处于、经过 (+0.2 each)

  4步智能判断流程:
    step_1: 功能性结构判断（人物介绍/评价？）
    step_2: 语义核心判断（删除后语义损失？）
    step_3: 冗余度判断（周围有哪些冗余修饰？）
    step_4: 自然性检查
```

### 原则3: 作者情感表达保护

```yaml
作者情感表达保护:
  定义: "区分AI空洞情感 vs 作者真实情感"

  保留:
    - 自然口语化："你想想"、"结果呢?"、"傻眼了"
    - 作者主观评价："可惜"、"遗憾"、"令人欣慰的是"
    - 个人声音："奇怪的是"、"倒是..."
    - 具体细节和人名

  删除:
    - 程序化评价："大概是...最..."、"太...了"
    - 陈词滥调："炸了锅"、"满座皆惊"
    - 功能性概括："这个角色体现了..."
```

### 原则4: 从分析模式到叙述模式

```yaml
终极改写原则: 从分析模式到叙述模式

AI的写作模式 = 分析模式❌:
  - 分析诗词与情节的关系
  - 概括诗词的作用
  - 总结诗词的特点
  - 分析文章的效果

人类的写作模式 = 叙述模式✅:
  - 讲述诗词如何融入故事
  - 描述诗词如何塑造人物
  - 叙述诗词如何表达情感
  - 表达诗词带来的感受

终极判断标准:
  这句话听起来像AI在分析，还是人类在讲故事？
  → 像AI在分析 → 彻底改写
  → 像人类在讲故事 → 保留
```

## 📊 改写强度分级

```yaml
Level 1: 轻量级改写（保守）
  适用: AIGC < 0.50
  策略: 删除AI高频词、程序化表达、微调语序
  预期效果: AIGC降低5-10%
  信息保留率: > 90%

Level 2: 中量级改写（标准）
  适用: AIGC 0.50-0.70
  策略: 拆解对称排比、替换功能式动词、重组段落结构
  预期效果: AIGC降低15-25%
  信息保留率: > 80%

Level 3: 重量级改写（激进）
  适用: AIGC > 0.70 或朱雀检测 > 0.80
  策略: 机械枚举→段落式叙述、删除高AIGC段落、完全重组语言节奏
  预期效果: AIGC降低30-50%
  信息保留率: > 70%（最低容忍度）
  注意: 最后手段，必须警告用户可能降低质量
```

## ⚠️ 质量检查机制

### 改写质量检查清单

```yaml
rewrite_quality_checks:

  check_1: 信息量检查
    metric: 信息保留率必须 > 80%
    test: 改写后是否丢失了重要信息？

  check_2: 流畅性检查
    metric: 流畅性必须 > 0.8（硬约束）
    test: 读起来是否流畅自然？

  check_3: 句式变化检查
    metric: 相同句式连续出现 < 2次
    test: 是否有句式重复？

  check_4: 自然性检查
    metric: 自然性必须 > 0.8
    test: 读起来像人类写作吗？
```

### 失败处理机制

```yaml
failure_handling:

  IF 信息保留率 < 70%:
    → 停止改写
    → 恢复原文或减少改写强度

  IF 流畅性 < 0.8:
    → 停止改写
    → 恢复必要语法词和连接词

  IF 句式重复 > 2次:
    → 重新改写
    → 打破句式一致性

  IF 自然性 < 0.8:
    → 调整改写方式
    → 保留更多自然表达
```

## 📋 AI高频词列表

### 极强特征（必须删除）

```yaml
极强AI高频词:
  - "下意识" → 删除，直接写动作
  - "渐渐"、"逐渐"、"缓缓" → 删除或改为"慢慢"
  - "连连"、"频频"、"纷纷" → 直接写动作
  - "静静"、"默默"、"悄悄" → 让动作说话
  - "猛然"、"骤然"、"突然" → 调整句式
  - "不由得"、"忍不住"、"情不自禁" → 直接写反应
  - "仿佛"、"好像"、"如同" → 直接描写或用破折号
```

### 强特征（建议替换）

```yaml
强AI高频词:
  - "果然"、"竟然"、"居然" → 删除或改为"确"
  - "展现了"、"体现了" → 尽显、凸显、彰显，或用破折号
  - "这是" → 此乃、该、或用破折号
  - "他们" → 删除（主语明确时）
```

### 功能式动词

```yaml
功能式动词:
  High-level (+0.4): 成为、符合、体现、承载、完成、实现
  Mid-level (+0.3): 培养、放弃、处理、管理、带领、发展、对抗、安排、展现、教导、增加、获得
  Basic-level (+0.2): 擅长、担任、负责、处于、经过
```

## 🚫 常见错误

### 错误1：过度删减

```yaml
问题: 为了降AIGC而过度删减，破坏句子自然性

表现:
  - 两个名词直接连用："那汉子张荣"
  - 动词宾语不完整："统领梁山泊水匪"
  - 缺少谓语："他眉头紧锁"
  - 破句感：读起来感觉少了什么

修复:
  - ✅ 优先使用"语序调整"而非"删减词汇"
  - ✅ 保留必要的语法结构（"是"、"的"）
  - ✅ 读一遍改写后的文本，检查是否自然
```

### 错误2：机械删除功能式动词

```yaml
问题: 机械删除所有功能式动词，不考虑语义必要性

表现:
  - 检测到"培养"、"处理"、"管理"等功能式动词就直接删除
  - 导致语义损失，失去人物主动性
  - 把功能性结构误判为AI特征

修复:
  - step_1: 功能性结构判断（人物介绍/评价？）
  - step_2: 语义核心判断（删除后语义损失？）
  - step_3: 冗余度判断（周围有哪些冗余修饰？）
  - step_4: 自然性检查
```

### 错误3：删除口语化表达

```yaml
问题: 删除口语化表达，反而增加AIGC值

表现:
  - 删除"你想想"、"结果呢?"、"傻眼了"
  - 删除作者的个人声音
  - 这些是强人类特征（ContentAny验证：AI 0.8%）

修复:
  - ✅ 保留自然口语化表达
  - ✅ 保留作者个人声音
  - ✅ 区分程序化评价 vs 自然口语化
```
