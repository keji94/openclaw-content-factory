# SOP_CONTENT.md · 内容生产标准流程

## 流程概览

```
素材入库&选题生成 → 选题推荐 → 大纲生成 → 初稿写作 → 智能审稿 → 润色打磨 → 定稿归档 → 归档发布
```

---

## ⚠️ 执行强制检查点

完成每个场景后，必须自问：

1. **当前场景是否完成？**
2. **SOP 里定义的下一个场景是什么？**
3. **是否需要自动触发下一个场景？**

**禁止在 SOP 定义的流程未完成前，向用户反馈「完成」。**

**特别说明**：
- 场景4（初稿写作）完成后，必须自动触发场景5（智能审稿）
- 场景5（智能审稿）完成后，必须自动触发场景6（润色打磨，调用 humanizer-zh）
- 只有场景6完成后，才能向用户反馈「初稿完成，请审阅」

---

## 场景匹配索引

匹配到场景后，**仅读取对应的场景文件**，不要全部加载。

| 触发条件 | 场景 | 场景文件 |
|---------|------|---------|
| 消息包含链接 | 场景1：素材入库&选题生成 | `SOP_SCENE_1_素材入库.md` |
| 用户说「推荐选题」「有什么选题」 | 场景2：选题推荐 | `SOP_SCENE_2_选题推荐.md` |
| 用户说「选1」「写第一个」 | 场景3：大纲生成 | `SOP_SCENE_3_大纲生成.md` |
| 用户选择大纲方向 | 场景4：初稿写作 | `SOP_SCENE_4_初稿写作.md` |
| 场景4完成后自动触发 | 场景5：智能审稿 | `SOP_SCENE_5_智能审稿.md` |
| 场景5完成后自动触发 | 场景6：润色打磨 | `SOP_SCENE_6_润色打磨.md` |
| 用户说「定稿」「完成了」 | 场景7：定稿归档 | `SOP_SCENE_7_定稿归档.md` |
| 用户选择发布到公众号 | 场景7.5：发布到公众号 | `SOP_SCENE_7.5_发布公众号.md` |
| 用户确认归档 | 场景8：归档发布 | `SOP_SCENE_8_归档发布.md` |

## 异常处理

| 异常情况 | 处理方式 |
|---------|---------|
| 用户中途取消 | 更新阶段为「已取消」，记录原因 |
| 需要补充信息 | 主动询问，记录到 MEMORY.md |
| 权限不足 | 明确告知需要开通的权限 |
| 搜索无结果 | 提示用户提供更多信息或参考 URL |
| 参考文章无法访问 | 跳过该 URL，继续处理其他素材 |
| 初稿质量过低 | 重新生成或提示用户提供更多素材 |
| 审稿评分 < 70 | 建议用户检查主题是否合适 |
