# 场景1：素材入库&选题生成

## 触发条件
用户发送的消息中包括链接或内容片段。

## 执行步骤

**Step 1: 内容抓取**
- 抖音/小红书/B站 → 使用 playwright-scraper 或对应 skill
- 公众号文章 → 使用 playwright-scraper 隐私模式
- 普通网页 → 使用 web_fetch
- 抓取成功后通知用户

**Step 2: 信息提取**
- 标题（原文标题）
- 来源（识别平台）
- 原文链接
- 内容摘要（200-400字核心要点）
- 标签（2-3个合适标签）
- 逐字稿：**如果是视频解析，原视频的逐字稿必须完整保存，一字不省** ⚠️ 逐字稿是花费 token 提取的有价值内容，是后续写作的原始素材

**Step 3: 选题拆解（核心）**
**强制要求**：从一篇素材拆解出 1-3 个不同角度的选题，每个选题一条记录。

选题拆解原则：
- 不同受众视角（新手 vs 进阶 vs 专家）
- 不同内容形式（教程 vs 观点 vs 案例）
- 不同切入点（痛点型 vs 方法型 vs 趋势型）

**选题价值模板**：
```
【选题标题】：xxx
【目标受众】：xxx
【核心价值】：xxx
【差异化】：与其他同类内容的不同点
【时效性】：是否需要紧跟热点
```


**Step 4: 写入飞书**

先获取字段类型（`feishu_bitable_app_table_field.list`），按格式写入：

```json
{
  "标题": "原文标题",
  "来源": "抖音/小红书/公众号/网页",
  "原文链接": {"link": "URL", "text": "查看原文"},
  "内容摘要": "AI提炼的摘要",
  "标签": ["#选题", "#素材"],
  "状态": "待处理",
  "优先级": "高/中/低",
  "选题标题": "从原文拆解的具体选题（必填）",
  "选题价值": "做这个选题的理由、价值、受众分析（必填）",
   "逐字稿": "如果是视频，必须保存逐字稿"
}
```

**Step 5: 反馈用户**
```
✅ 已从原文拆解出 X 个选题

📌 原文标题：xxx
🏷️ 标签：#选题 #素材

💡 拆解出的选题：
1. 【选题标题1】（优先级：高）
   受众：xxx | 价值：xxx
2. 【选题标题2】（优先级：中）
   受众：xxx | 价值：xxx

共入库 X 条选题记录。
```
